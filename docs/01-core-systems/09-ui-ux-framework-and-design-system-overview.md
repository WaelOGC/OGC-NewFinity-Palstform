# OGC NewFinity — UI/UX Framework & Design System Overview (v1.0)



## 1. Introduction

This document defines the full UI/UX framework and design system used across the OGC NewFinity ecosystem.  

It establishes the core aesthetic, interaction principles, color system, typography, layout structure, and reusable components that shape every dashboard, page, and workflow.



This is the master reference for all UI/UX designers, frontend developers, and branding teams.



---



# 2. Design Philosophy



### 2.1 Core Principles

- **Futuristic** — AI-powered, next-generation aesthetic  

- **Clean & Minimal** — no clutter, clear hierarchy  

- **Plasma Identity** — signature OGC color palette with fluid gradients, soft radiance, and molten-style transitions  

- **Dark Theme First** — transparent layers, glowing accents  

- **Consistency** — same components across all dashboards  

- **Accessibility** — readable contrast, logical structure  

- **Scalability** — new tools and dashboards can be added easily  



### 2.2 Experience Goals

- Intuitive navigation  

- Clear role-based visibility  

- Smooth animations  

- Zero cognitive overload  

- Efficient user flows  

- Modern, premium tech-platform feel  



---



# 3. Color System



## 3.1 Core Brand Colors

- **Plasma Teal** — `#00FFC6` (fluid gradient with soft radiance)  

- **Sunburst Yellow** — `#FFBC25`  

- **Deep Violet Blue** — `#5864FF`  

- **Electric Pink** — `#FF3CAC`



## 3.2 Supporting Colors

- White (for text only)  

- Light Gray: for borders/dividers  

- Transparent overlays for depth effects  



## 3.3 Usage Guidelines

- **White text only** — consistent across the ecosystem  

- **Plasma colors** (fluid gradients, soft radiance, molten-style transitions) used for:

  - Highlights

  - Buttons

  - Active states

  - Key accent elements  

- **Black/dark backgrounds** for contrast  

- Transparent panels to reflect a high-tech interface  



---



# 4. Typography



### 4.1 Primary Font

- **Inter** (default)  

- Clean, modern, highly readable  



### 4.2 Typographic Scale

- H1 — 34–42px  

- H2 — 26–32px  

- H3 — 20–24px  

- Body — 14–18px  

- Label — 12–14px  



### 4.3 Rules

- No serif fonts  

- Consistent spacing between lines  

- Minimal use of bold styles  



---



# 5. Grid & Layout Structure



### 5.1 Grid System

- 12-column responsive grid  

- 24px outer margins  

- 16–20px internal spacing  

- Heavy use of spacing tokens (8px multiples)



### 5.2 Panel Structure

- Floating panels with blur  

- Card-based layout  

- Rounded corners (6–10px)  

- Subtle plasma internal glow with diffused depth  



### 5.3 Navigation Layout

- Sidebar left (platform + wallet + Amy dashboards)  

- Top header: notifications, token balance, profile  

- Content area: flexible, centered grid  



---



# 6. Component System (UI Kit)



## 6.1 Buttons

- **Primary Plasma Button** (smooth edges, internal glow)  

- **Outline Button**  

- **Ghost Button**  

- **Danger Button**  



Rules:

- Rounded corners  

- Hover glow  

- Smooth transitions  



---



## 6.2 Cards

- Transparent layered cards  

- Plasma border with internal glow and diffused depth  

- Icon or illustration support  

- Used in Dashboards, Challenges, Wallet, AI Tools  



---



## 6.3 Forms & Inputs

- Transparent backgrounds  

- White text  

- Subtle plasma internal glow on focus  

- Error state in red glow  



---



## 6.4 Modals

- Dimmed background  

- Centered container  

- Animated fade-in  

- Two-button actions  



---



## 6.5 Sidebar Navigation

- Icons + labels  

- Glow on hover  

- Active state plasma highlight with soft pulsing  

- Responsive collapse mode  



---



## 6.6 Tables

- Minimal borders  

- Alternating row transparency  

- Highlight on hover

## 6.7 Notifications UI

# 7. Interaction & Motion Guidelines



### 7.1 Hover Effects

- Glow or color shift  

- Smooth transitions (150–250ms)



### 7.2 Page Transitions

- Subtle fade or slide  

- No flashy animations  



### 7.3 Component Animations

- Button press feedback  

- Card hover expansion  

- Modal fade in/out  



### 7.4 Loading States

- Plasma progress rings with wave motion  

- Skeleton cards

# 9. Accessibility



### Requirements

- Minimum contrast ratio for text  

- Keyboard-friendly navigation  

- Proper ARIA roles  

- Font sizes ≥ 14px  



---



# 10. Multi-Dashboard Consistency



All dashboards must use the same:

- Colors  

- Typography  

- Layout spacing  

- Button styles  

- Cards  

- Icons  

- Motion effects  



This ensures a unified OGC brand across:

- Core Platform  

- Wallet Dashboard  

- Amy Agent Dashboard  

- Admin Panel

# Standardized Roadmap Reference

This document provides the official 6-phase roadmap for OGC NewFinity that should be used across ALL documentation.

## Official Roadmap Diagram

See: `roadmap-standardized.mmd`

## Phase Breakdown

### Phase 1 — Genesis Deployment (Complete)

- OGCFinity Genesis Token launched on Polygon (ERC-20)
- Wallet integration + tokenomics finalization
- Contribution Mining Pool activated

### Phase 2 — Core Platform (In Progress)

- AI Agent Amy (core features)
- Dashboards + submission portal
- Token-gated utilities using OGCFinity

### Phase 3 — Challenge Program Activation

- Launch OGC NewFinity Challenge Program
- Three official tracks
- First grants + public transparency reporting

### Phase 4 — Platform Expansion

- Developer SDKs
- Marketplace (tools, datasets, models)
- OGC Vault (native wallet with staking)

### Phase 5 — Governance Layer

- DAO governance
- Token voting
- Treasury oversight

### Phase 6 — OGC Chain (Future Vision)

- Native blockchain (OGC Chain)
- OGCFinity Native Token (governance-based supply)
- Secure optional 1:1 migration bridge

## Usage

When creating or updating any roadmap documentation:
1. Use this exact phase structure
2. Reference the standardized diagram
3. Maintain consistent terminology:
   - "OGCFinity" (not "OGC Token")
   - "Contribution-Based Mining" (not "contribution mining")
   - "OGC Vault" (future evolution of OGC Wallet)
   - "OGC Chain" (future blockchain)

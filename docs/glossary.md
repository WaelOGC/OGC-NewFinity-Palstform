# OGC NewFinity Platform — Glossary of Terms

This glossary defines all key terms, systems, and concepts used throughout the OGC NewFinity Platform documentation.

---

## A

### Access Token

A short-lived JWT used to authenticate user actions.

### Admin Dashboard

The control panel used by administrators to manage users, challenges, rewards, badges, and system settings.

### Amy AI Agent

The platform's AI engine responsible for tools like writing, code assistance, automation, design, and content generation.

---

## B

### Badge Engine

Subsystem that calculates and awards badges based on user achievements and milestone triggers.

### Blockchain Integration

Module that interacts with the Polygon network for syncing balances and executing transactions.

---

## C

### Challenge

A user-submitted task or competition entry within the Challenge Program.

### Cursor Agent

The automated development agent running inside Cursor IDE, responsible for applying naming rules, creating files, and maintaining consistency.

---

## D

### Dashboard

Any main interface screen for Platform, Wallet, Amy, or Admin.

### Data Model (ERD)

The diagram representing database entities and relationships.

---

## E

### Escalation Flow

Governance process for handling user reports, violations, and appeals.

---

## F

### Frontend

React + Vite codebase (planned) powering the user interface.

---

## G

### Governance

System of rules, moderation, permissions, escalation, and compliance actions.

---

## H

### History

User's stored outputs, saved results, and tool activity logs.

---

## I

### Index File

A documentation entry point listing files within a folder (e.g., docs/mermaid/index.md).

---

## M

### Mermaid Diagram

A flowchart, sequence diagram, or architecture diagram written in Mermaid syntax.

---

## P

### Phase

Documentation grouping that organizes diagrams by system evolution (Phase 1–9).

### Prisma

ORM used to manage the database schema and migrations.

---

## R

### Rewards Engine

Calculates user-earned tokens from challenges, badges, contributions, and events.

### Reconciliation

Process that compares on-chain vs off-chain wallet data.

---

## S

### Sequence Diagram

A Mermaid diagram showing interactions between services or actors step-by-step.

### Subscription

User's paid access to premium platform features.

---

## T

### Token

The OGC utility token used for rewards and platform interactions.

### Transaction Processor

Wallet subsystem that validates and executes user-initiated transactions.

---

## U

### Unified Master Flow

A single diagram showing how all platform systems connect across major user journeys.

### User Flow

Diagram showing how a user moves through key parts of the platform.

---

## W

### Wallet Dashboard

User interface for token balance, transactions, rewards, and blockchain syncing.

### Wallet Service

Backend module powering the wallet system, transaction logic, rewards logic, and syncing engine.

---

End of glossary.

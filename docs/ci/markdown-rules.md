# Markdown Validation Rules

These rules define how Markdown files must be validated.

---

## 1. Syntax Checks

- All files must follow standard Markdown formatting.

- Headings must use #, ##, ### properly.

- Lists must be consistent.

- Code blocks must be properly enclosed.

---

## 2. Lint Requirements

Future CI will run:

- markdownlint

- prettier formatting

- whitespace checks

---

## 3. Common Errors to Avoid

- Unclosed code blocks

- Missing line breaks after headings

- Invalid indentation

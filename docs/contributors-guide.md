# OGC NewFinity — Documentation Contributors Guide

This guide defines how contributors should update and maintain the documentation for the OGC NewFinity Platform.

---

## 1. General Rules

- Write clearly and concisely.  
- Maintain consistent tone and formatting.  
- Update all relevant index files when adding new content.  
- Do not leave placeholders or incomplete sections.  
- Follow the existing folder and naming structure.

---

## 2. Mermaid Diagram Standards

All diagrams must follow:

phase-X[.Y]--category--topic.mmd

When adding a new diagram:

1. Place it in the correct phase folder.  
2. Update the folder's README.  
3. Add a link to docs/mermaid/index.md.  
4. Follow spacing and naming consistency.  

---

## 3. Updating Documentation

When changing or adding documentation:

- Update docs/index.md if it's a major addition.  
- Update docs/documentation-overview.md for key files.  
- Add entries to docs/shortcuts.md if widely used.  
- Reflect directory changes in docs/structure-map.md.  

---

## 4. Using Cursor Agent

All contributors MUST use:

- .cursor/context/naming-rules.md  
- .cursor/context/agent-guidelines.md  

These rules ensure naming and formatting consistency across the project.

---

## 5. Do Not Break These Rules

- Do NOT create new folders without approval.  
- Do NOT create diagram names outside the naming pattern.  
- Do NOT mix unrelated content in the same file.  
- Do NOT leave unused or empty documentation files.  

---

## 6. Before Submitting Changes

Contributors must:

- Validate Markdown syntax  
- Ensure no lint errors  
- Confirm links are working  
- Ensure naming rules are followed  

---

## 7. Contact

For questions or approval, contact the OGC Technologies documentation owner.
